### Q1. What is the primary purpose of a README.md file in a repository?
- A) To list all contributors
- B) To provide an overview, setup instructions, and documentation for the project
- C) To define branch protection rules
- D) To configure CI/CD pipelines

### Q2. Which file specifies the legal terms under which a project's code can be used?
- A) CONTRIBUTING.md
- B) CODEOWNERS
- C) LICENSE
- D) README.md

### Q3. What does the CODEOWNERS file do?
- A) Lists users who starred the repository
- B) Automatically assigns reviewers to pull requests based on file paths
- C) Grants admin access to listed users
- D) Defines the repository's license

### Q4. When creating a new repository on GitHub, which of the following can be auto-generated?
- A) GitHub Actions workflows
- B) README, .gitignore, and LICENSE files
- C) Branch protection rules
- D) Repository webhooks

### Q5. What is the purpose of a .gitignore file?
- A) To ignore specific branches during merges
- B) To specify files and directories that Git should not track
- C) To block certain users from contributing
- D) To hide the repository from search results

### Q6. What is the difference between a Git tag and a GitHub release?
- A) They are identical features
- B) A tag marks a specific commit; a release is a GitHub feature built on tags that can include release notes and binary assets
- C) Tags are for branches; releases are for commits
- D) Releases are only available on Enterprise plans

### Q7. What does the "Insights" tab of a GitHub repository provide?
- A) Repository billing information
- B) Analytics including contributor activity, traffic, commit frequency, and dependency graphs
- C) A list of all repository forks
- D) Security vulnerability reports only

### Q8. What is a repository template on GitHub?
- A) A repository that cannot be modified
- B) A repository that others can use as a starting point to generate new repositories with the same structure
- C) A private repository visible only to organization owners
- D) A repository that automatically syncs with forks

### Q9. Which file provides guidelines for how others should contribute to a project?
- A) README.md
- B) LICENSE
- C) CONTRIBUTING.md
- D) CODEOWNERS

### Q10. What is the default visibility when creating a new repository under a free personal GitHub account?
- A) Internal
- B) Private (you must explicitly choose public)
- C) Public (you must explicitly choose private)
- D) You choose between Public or Private during creation
